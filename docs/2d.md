# 2D 渲染器设计

## 屏幕2D

### 空间结构管理

#### 2D Unit

- id
- 2D AABB：所有 glyph 范围的包围盒
- z-index
- layer: 所属layer，由Layer系统分配

#### Layer

每个 Layer 包含若干 Unit，对任意两个2D Unit，满足以下两个条件之一：
- z-index 相同
- z-index 不同，且AABB标识的范围没有重叠

每个layer提供查询功能：
- 给一个坐标，查询坐标是否命中一个unit
- 给一个 AABB，查询范围内重合的unit

#### Layer 管理

- 增加一个 layer
- 向数据结构中插入一个 unit
- 从layer 移除 unit


### GPU 资源和渲染

#### 描述符集

- binding 0: 静态字符 atlas
- binding 1: 动态字符 atlas
- binding 2: glyph buffer的不定长数组

#### Glyph 结构

- vec4 quadRect 左下角和右上角坐标
- vec4 uvRect 左下角和右上角uv
- vec4 color 颜色或其他浮点信息
- uvec4 info 整型信息
  - x: texture id

共占用4 * 4 * 4B = 64B 

#### Glyph Manager

开辟若干大的buffer作为所有2D glyph共有的存储空间

- 一份CPU存储glyph的CPU状态
- 每个buffer固定包含512个glyph，最多16个buffer，总容量8192个glyph
- MAX_FRAMES_IN_FLIGHT 份 device buffer，每一帧从CPU buffer同步当前帧的buffer
- 每个buffer独立使用 updateCnt 计数，修改后设为 MAX_FRAMES_IN_FLIGHT，每次整体同步对应buffer后updateCnt -= 1
- 维护一个空闲列表，空闲列表记录glyph buffer里的空闲槽位编号
  - 分配时从空闲列表里找空闲槽位，返回buffer编号和buffer内槽位编号组成的一个uint32_t
  - 不够分就开一个新的buffer和 MAX_FRAMES_IN_FLIGHT 个 device buffer
  - 回收时接受编号列表，放回空闲列表里
- 在描述符集里用 non uniform 维护 buffer描述符，有 MAX_FRAMES_IN_FLIGHT 份描述符集，同样用一个 updateCnt 控制更新

#### 2D Render Unit

与空间管理的 2D Unit 对应

- id：2D Unit的id
- model matrix：对transform模型矩阵的指针
- glyph ids：包含的所有 glyph

渲染时，绑定顶点数组（所有glyph id），设定 model 的 push constant 进行GPU实例渲染。不同unit之间不合批，每个unit独立产生一次draw call。

#### 2D Render Info

类似3d的render info，表示共用一个material的所有unit

- material：共用的材质
- units：所有 2D Render Unit 的unordered map，按id索引

支持以下操作：

- 插入unit：插入unit到map里，触发更新
- 删除unit：从map里删除unit，触发更新
- 渲染：时绑定材质pipeline和描述符集，遍历unit进行渲染


#### 2D Render Layer

包含若干 2D Render Info，按材质索引

支持以下操作：

- 插入unit：按材质找到或创建info进行插入
- 删除unit：按材质找到info进行删除

#### Layered 2D Renderer

用于半透明等需要混合的2D渲染，与空间管理的 Layer 对应，渲染的时候从后向前遍历layer渲染。

支持以下操作：

- 分配 unit：以 unit id 作为输入，从 Glyph Manager 里分配glyph构造unit
- 销毁 unit：以 unit id 作为输入，触发 Glyph Manager 回收unit
- 更新 unit 的 glyph：向 Glyph Manager 标记更新
- 向 layer 添加 unit：以 unit id 和 layer id 作为输入，向layer添加unit
- 从 layer 移除 unit：以 unit id 和 layer id 作为输入，移除layer中的unit

  
这些操作的合法性不需要在这个层面检查，上层会有检查


### UI 组件基类

一个entity对应一个，与 renderable object类似，使用entity对应的transform的model矩阵作为变换矩阵，z作为z-index

- id：从 UI 空间结构里分配的id
- material：使用的材质
- 2D render unit

由组件的生命周期来管理相关空间和渲染资源的分配和释放
